December 7, 2025

Basement OS Web Terminal Launch

I finally got the web terminal up and running! This has been on my mind for a while - having a way to interact with Basement OS outside of VRChat. The idea is to mirror what you'd see on the CRT monitor in the basement, but accessible from any browser.

Took some inspiration from HumanLayer'sA company building tools for AI agents that need human oversight blog - they have this clean theming system I really liked. Implemented 13 different color themes, but keeping High Contrast as the default because readability comes first.

The boot sequence mirrors what happens in the actual VRChat world. When you approach the terminal in the basement, you see the same POST checks and loading messages. It's all about that nostalgic 2000s feel.

December 6, 2025

MILESTONE: Closed-Loop Agent System

This is a big one. I've been working with Claude CodeAn AI coding assistant by Anthropic that can read, write, and execute code autonomously to build out the Basement OS kernel, and we finally cracked the automation problem.

The issue with UdonSharpA C# to Udon compiler that lets you write VRChat scripts in familiar C# syntax instead of visual programming is that you can't just write code - you need Unity to generate .asset files, attach them to GameObjects, and compile everything. Previously this meant constant back-and-forth with the human (me) clicking buttons in Unity.

Now with Unity MCPModel Context Protocol - a way for AI agents to communicate with Unity Editor directly, Claude can do the full loop: write the script, trigger compilation, check for errors, attach to GameObjects, enter Play mode, and verify everything works. No human intervention needed.

The hub-spoke architectureA design pattern where a central 'hub' (DT_Core) routes messages to multiple 'spoke' components (apps) is working beautifully. DT_Core handles all the hardware stuff (screen, input, audio), and apps in /BIN/ just do their thing without worrying about low-level details.

November 15, 2025

Terminal Menu System Complete

Replaced the old auto-cycling display with an actual interactive menu. Players can now use their movement controls to navigate up/down through options and select with the interact button.

Had to implement player immobilization when they're at the terminal - otherwise pressing up/down would move your avatar AND the cursor. Using VRCStationA VRChat component that locks a player in place and captures their input for this, which also solves the "walking away mid-interaction" problem.

Also extracted the weather module into its own script. The terminal now pulls real-time weather data from my GitHub PagesFree static website hosting from GitHub - I use it to serve weather data as JSON endpoint and displays it in the header. When it's actually raining in Fond du Lac, you'll see rain in the basement too.

October 20, 2025

MILESTONE: Achievement System Overhaul

Finally finished the Xbox 360-style achievement system! 19 achievements worth 420G total (nice). The notifications pop up just like they did on the 360 - that satisfying bloop sound and the sliding banner.

Using VRChat's PlayerData APIVRChat's persistence system that saves player data across sessions - limited to basic types like int, float, string for persistence. This was tricky because you can't use fancy C# features in UdonSharpA C# to Udon compiler - no generics, no LINQ, no async, no try/catch - no List<T>, no Dictionary, no LINQ. Everything's done with arrays and careful indexing.

The FIFO queueFirst-In-First-Out - notifications appear in the order they were earned, like a live feed for notifications took a few iterations. Originally had a priority system but it felt weird when achievements popped up out of order. The chronological approach matches the "basement live feed" vibe I was going for.

Hey, I'm CloudStrife7. This is my dev blog for Lower Level 2.0 - a VRChat world I've been building that recreates a 2000s basement gaming setup.

The project started as a simple hangout space but evolved into something more ambitious. Now it's got an Xbox 360-style achievement system, a fully interactive DOS terminal (that's Basement OS), real-time weather integration, and a bunch of nostalgic touches.

I use this blog to document my progress, share technical solutions, and explain the weird workarounds needed when working with UdonSharpVRChat's scripting system - a subset of C# with many limitations. If you're building VRChat worlds, hopefully some of this is useful.
